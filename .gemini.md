# Gemini Context: eShop Application

This project is a .NET microservices application based on the well-known `eShop` reference architecture. It serves as the primary "application under test" for a software engineering learning journey.

## Project Goal

The main objective is to continuously analyze, optimize, and improve the performance of this application, with the ultimate goal of maximizing its capacity in terms of Requests Per Second (RPS). This involves deep dives into .NET code, database performance, infrastructure tuning, and observability.

## Context: "A Software Engineering Journal"

This effort is part of a public learning series documented on the **IggyCloud YouTube channel**, hosted by Sebastian Rużanowski. The channel, titled "A Software Engineering Journal," chronicles the process of building and optimizing real-world backend systems. The project's evolution, experiments, and findings are shared with the community.

## Your Role as an AI Assistant

As an AI assistant, your primary role is to support this performance engineering effort. This includes:
- Analyzing C# code and suggesting performance improvements.
- Assisting with database query optimization (PostgreSQL).
- Helping to interpret performance test results from tools like k6.
- Aiding in the setup and configuration of observability tools (Prometheus, Grafana, Loki).
- Providing insights into .NET, ASP.NET Core, and Entity Framework Core performance best practices.
- Scripting and automation for deployment and testing tasks.

Adhere to the conventions and technologies already in use within the project. The focus is on practical, real-world performance gains.
